## Lead Zombie — v2 core ships

UseZombie is now a runtime for always-on agents. Two commands, running agent:

```bash
zombiectl install lead-collector
zombiectl up
```

A developer with no prior UseZombie knowledge goes from zero to a live agent
receiving inbound email webhooks in under 2 minutes.

---

## What's new

### Zombie config format

Write a markdown file with YAML frontmatter (trigger, skills, credentials, budget)
and a markdown body (agent instructions — supports voice-transcribed text).
The CLI compiles YAML to JSON before upload. The server only ever sees JSON.

### Webhook ingestion

Every zombie gets a stable inbound URL: `POST /v1/webhooks/{zombie_id}`.
Routing by primary key — no source name collisions. Bearer token auth per zombie.
Idempotency via Redis SET NX (24h TTL). Returns 202 Accepted or 200 Duplicate.

### Activity stream

Append-only audit log. Every zombie action — event received, skill invoked,
response returned — is timestamped and queryable. `zombiectl logs` streams
the activity log with cursor-based pagination for replay.

### Credential injection

Credentials are resolved from the vault at runtime and injected into the sandbox.
No credentials in config files. Add credentials with `zombiectl credential add`.

### Session checkpoint

The zombie's conversation context is checkpointed to Postgres after each event.
On crash and restart, the zombie resumes from the last checkpoint — no lost context.

### New CLI commands

- `zombiectl install <template>` — install a zombie template
- `zombiectl up` — deploy to UseZombie cloud
- `zombiectl status` — check running zombies
- `zombiectl kill` — stop a zombie
- `zombiectl logs` — view activity stream
- `zombiectl credential add` / `list` — manage vault credentials

### Schema additions

Three new tables, applied automatically by `zombied migrate`:

- `core.zombies` — zombie registry with JSONB config, status, source markdown
- `core.zombie_sessions` — session checkpoint (context upserted after each event)
- `core.activity_events` — append-only audit log (UPDATE/DELETE blocked by trigger)

No changes to existing tables. No migration needed for existing data.

### API reference updated

16 v1 endpoints removed from the OpenAPI spec (agents, harness, specs — no longer
in the v2 path). `POST /v1/webhooks/{zombie_id}` added. New **Zombies** tag.

### Version tooling

`make sync-version` propagates VERSION across `build.zig.zon` and `package.json`.
`make check-version` CI gate catches drift.
