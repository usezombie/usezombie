# v0.5.0

**Date:** Apr 08, 2026
**Milestone:** M1_001
**Spec:** M1_001_LEAD_ZOMBIE.md

## What changed

- **`zombiectl install <template>`** — new command that writes a Zombie config (markdown with YAML frontmatter) to the current directory. Bundled `lead-collector` template ships inside the npm package; works offline, no registry needed.
- **`zombiectl up`** — new command that deploys the local Zombie config to UseZombie cloud via API. Remote-first; Mac developers never need bwrap.
- **`zombiectl status`** — new command showing Zombie name, status, events processed, and budget used.
- **`zombiectl kill`** — new command that sends DELETE to decommission a running Zombie.
- **`zombiectl logs`** — new command fetching the activity stream for a Zombie.
- **`zombiectl credential add/list`** — new subcommands for storing and listing Zombie credentials via vault API.
- **Zombie event loop** (`src/zombie/event_loop.zig`) — `claimZombie`, `runEventLoop`, `deliverEvent`, `checkpointState`, `updateSessionContext`. Persistent agent process with at-least-once delivery (XACK after checkpoint), crash recovery from Postgres session checkpoint, and JSON-safe context injection.
- **Redis Zombie queue** (`src/queue/redis_zombie.zig`) — `ensureZombieConsumerGroup`, `xreadgroupZombie`, `xautoclaimZombie`, `xackZombie` — isolated from main queue module.
- **5 new error codes** — `UZ-ZMB-001` through `UZ-ZMB-005` with actionable hints (budget exceeded, agent timeout, credential not found, session checkpoint failed, worker state invalid).
- **19 new CLI unit tests** covering all zombie command dimensions (5.1–5.4) plus kill, logs, and unknown subcommand paths.
- **22 new Zig unit tests** for event loop internals plus 8 DB-gated integration tests for full-stack dims 3.1/3.3.

## Technical details

- `zombie.install` route added to `AUTH_EXEMPT_ROUTES` — users can install templates before logging in.
- `updateSessionContext` uses `std.json.Stringify.valueAlloc` for JSON-safe context injection (no allocPrint string interpolation).
- Status read from Postgres row happens before `q.drain()` to avoid use-after-free on `[]const u8` slice.
- `parsed_ctx` lifetime managed with `defer p.deinit()` to prevent leak on long-running event loop iterations.
- All new Zig files split to stay under 500-line CI gate: `redis_zombie.zig` (195L), `event_loop.zig` (443L), `event_loop_test.zig` (430L).
- Cross-compiles on `x86_64-linux` and `aarch64-linux` verified.
