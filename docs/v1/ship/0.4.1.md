# v0.4.1

**Date:** Apr 06, 2026
**Milestone:** M21_002
**Spec:** M21_002_INSTANT_INTERRUPT_DELIVERY.md

## What changed

- Interrupts sent with `mode: "instant"` are now delivered within ~2s while gate commands run, instead of waiting up to 60s for the next checkpoint poll
- Gate child process is killed on interrupt (SIGTERM → SIGKILL after 5s grace) so the agent absorbs the user's steer immediately
- New `zombie_interrupt_delivery_latency_ms` Prometheus histogram tracks interrupt delivery latency

## Technical details

- Option B (timer thread) implementation: a thread polls Redis every 2s during `executeGateCommand` for interrupt signals
- Timer thread uses CAS on an atomic exit reason to safely race with timeout and natural exit
- Redis client is thread-safe (mutex-guarded) — no separate connection needed
- Extracted `worker_gate_helpers.zig`, `worker_gate_limits.zig`, `metrics_histograms.zig`, `metrics_interrupt_histogram.zig` to keep all files under 500 lines
- `incInterruptInstant()` counter now has a live call site (was dead code in M21_001)
- Structured log: `gate_loop.interrupt_killed_gate` with `run_id`, `workspace_id`, `agent_id`, `delivery_mode`, `wall_ms`
