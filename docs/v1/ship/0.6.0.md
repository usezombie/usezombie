# v0.6.0

**Date:** Apr 10, 2026
**Milestone:** M2_001
**Spec:** M2_001_LEAD_ZOMBIE_E2E.md

## What changed

- Zombie CRUD API: create, list, delete zombies via `POST/GET /v1/zombies/`, `DELETE /v1/zombies/{id}`
- Activity stream endpoint: `GET /v1/zombies/activity` with cursor-based pagination
- Credential management: `POST/GET /v1/zombies/credentials` with envelope encryption via vault.secrets
- Worker integration: zombie threads spawn at worker boot, 1:1 zombie-to-thread mapping
- Credential injection: event loop resolves credentials from vault and passes to executor
- Network allowlist: `api.agentmail.to` added for agentmail skill
- Sample workspace: `samples/lead-collector/` reference implementation
- ZombieStatus enum: type-safe status transitions (active, paused, stopped, killed)

## Technical details
