## What changed

- Added schema compatibility mode to `zombied doctor` with `--schema-gate --format json`, including machine-parseable reason codes.
- Deploy verification now relies on Fly.io release-command migration flow instead of an SSH-based schema probe in DEV CI.
- Documented operator triage for schema compatibility checks and post-fix validation.

## Technical details

- Schema compatibility compares binary migration set (`common.canonicalMigrations()`) with database state (`audit.schema_migrations`) via `db.inspectMigrationState`.
- Schema gate fails on pending, failed, or schema-ahead migration states and reports `SCHEMA_*` reason codes.
- DEV and PROD Fly app configs use `release_command = "/usr/local/bin/zombied migrate"`.
