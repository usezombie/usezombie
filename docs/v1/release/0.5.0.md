# v0.5.0

**Date:** Apr 05, 2026
**Milestone:** M31_001
**Spec:** M31_001_SCHEMA_STANDARDIZATION.md

## What changed

- Standardized core schema foundations by splitting the old monolithic `001_initial.sql` into focused migration units (`001_core_foundation.sql`, `002_core_workflow.sql`, `003_core_results_events.sql`).
- Standardized `core.platform_llm_keys` to application-generated UUIDv7 IDs and BIGINT millisecond timestamps.
- Added explicit schema conventions in `docs/contributing/SCHEMA_CONVENTIONS.md` for IDs, timestamps, mutable-table `updated_at`, and append-only audit patterns.

## Technical details

- Slid schema numbering to preserve ordered migration history (`004..021`) and updated compile-time embeds plus canonical migration array (`schema/embed.zig`, `src/cmd/common.zig`).
- Added `generatePlatformLlmKeyId()` and switched platform key write paths/tests to `std.time.milliTimestamp()`.
- Added `updated_at` to `core.tenants` and `core.workspace_memories`; enforced append-only behavior for `core.gate_results` with trigger-based UPDATE/DELETE rejection.
- Preserved `core.run_interrupts` in the split schema files to avoid regression from `001_initial.sql` removal.
- Verified with `make lint`, `make test`, `make build`, `make check-pg-drain`, and `make test-integration-db`.
