## 8 · If something breaks

1. **`make lint` fails:** read the output. Don't `--no-verify`. Investigate root cause. The user's operating model: "do not use destructive actions as a shortcut to simply make it go away."
2. **Vitest fails on an existing test:** lazy-loaded routes need `findBy*` not `getBy*` (lesson from commit `af443d2b`). React 19 behavior may have tightened some rules (`react-hooks/set-state-in-effect`, `react-hooks/refs`) — fix the antipattern, don't disable the rule.
3. **Playwright smoke fails:** run the build, grep the emitted CSS for the missing utility. If it's missing from the bundle, `@source` is likely broken. If it's in the bundle but computed-style is wrong, the theme token isn't bridged.
4. **Radix Slot error "React.Children.only expected to receive a single React element child":** the `asChild` branch can't have multiple siblings. See how `Card.tsx:57` handles this — two separate return paths (Slot branch without auto-badge, article branch with it).
5. **Stuck on a design call:** ask the user. Do not guess on aesthetic questions. The user explicitly wants `/design-shotgun` for M26.6.
