## 3. Functionally true on the branch right now

- `POST /steer` → direct XADD → `202 {event_id}`.
- Worker 13-step `processEvent` runs end-to-end (INSERT received → gates → executor → UPDATE terminal → telemetry → session bookmark → PUBLISH event_complete → XACK).
- Executor streams `tool_call_started` / `chunk` / `tool_call_completed` / `tool_call_progress` frames; `activity_publisher` PUBLISHes each to `zombie:{id}:activity`.
- `GET /events` returns durable history paginated newest-first with `actor` glob / `since` (duration or RFC3339) / `cursor` filters.
- `GET /events/stream` SSE forwards each PUBLISH frame as one SSE event with per-connection monotonic sequence ids.
- `zombiectl steer <id> "<msg>"` does end-to-end live tail (POST → SSE filter → fallback poll).
- Dashboard `/zombies/{id}` renders `<LiveEventsPanel />` above `<EventsList />` — browser uses native `EventSource` against same-origin `/backend/.../events/stream`, intercepted by a Next.js Route Handler that mints an api-audience JWT server-side via Clerk SDK and proxies the upstream stream with `Authorization: Bearer` injected. Backend SSE endpoint stays Bearer-only.
- `zombied-executor-harness` binary built alongside the production `zombied-executor`. Harness comptime-strips the entire NullClaw agent loop (1.7 MB vs. 22 MB binary size delta proves the gate is real). When `EXECUTOR_HARNESS_SCRIPT` env var is set, the harness reads a JSON script and emits the listed `ProgressFrame`s via the same `ProgressWriter` production uses.

## 9. Things this session learned (don't repeat)

- **`std.http.Client` (used by the test harness) cannot parse a 413 once the server closes the connection mid-body-write.** Body-size-limit tests need a raw-socket fixture. Affects `test_event_payload_size_limit`; deferred.
- **Clerk's `__session` cookie has no API audience claim.** Trying to validate it against the server's strict `aud` check 401s. The Next Route Handler approach sidesteps this entirely.
- **`@usezombie/design-system` does not export a List/Timeline primitive.** `EventsList` and `LiveEventsPanel` compose `Card` + `Badge` + `Pagination`. If you find yourself wanting a generic `<DataList>` again, spec out a design-system primitive workstream rather than re-implementing.
- **The harness binary stripping is real, not theoretical.** 22 MB → 1.7 MB on the executor binary when the comptime flag flips. Trust the gate — but if you ever need to verify, `du -h zig-out/bin/zombied-executor*`.
- **`bun run vitest --run` in `ui/packages/app/`** globs into `node_modules/`. Pre-existing config drift, unrelated to M42. Run targeted file lists when verifying changes (`vitest --run lib/api/events.test.ts dashboard-coverage coverage-edges`).
- **Pre-push hook is opinionated.** Runs `make lint` + multiple test targets before pushing. Don't rely on `zig build test` alone — the executor test target catches signature mismatches the worker build doesn't.
- **`var msg_opt = try …` triggers "local variable is never mutated"** in Zig 0.15.2. Use `const` when not reassigning.
- **The `Section` design-system primitive renders a `<div>` by default.** Use `asChild` to keep semantic `<section>` (per `<Section asChild><section aria-label="...">…</section></Section>` pattern, established this session).
- **AGENTS.md required-output formats are now compressed single-line backticks** (`LEGACY CONSULT: <desc> | found:<file:line> | …`). Same fields, denser prose. Self-audit greps at end-of-turn enforce them — they're not advisory.
