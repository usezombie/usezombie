## Decisions + surprises

- **Schema resequence.** User explicitly overrode the "slot gaps are fine"
  rule and asked for contiguous `001..018`. Renamed all SQL files, rewrote
  `schema/embed.zig` and `src/cmd/common.zig` (versions 1..18, length 18,
  last-version test updated to 18; `workspace_entitlements` test index flipped
  from 14 → 4). Caught two stale `-- NNN_filename.sql` header comments in
  renamed files and refreshed them.
- **PlanTier lives on `tenant_billing.zig`.** Old `workspace_billing_model.PlanTier`
  had the `{free, scale}` set plus Stripe-era ceremony. The new enum keeps the
  two-variant set so `metering.zig`'s `scale-tier exempt` branch still compiles;
  Stripe-tier expansion deferred to the Stripe-wiring milestone.
- **Replay idempotency is a known MVP gap.** Without an audit table, a
  redelivered zombie event can double-debit the tenant balance. Spec Out-of-Scope
  already lists audit tables as deferred; this is the trade-off.
- **`enforceFreeWorkspaceCreationAllowed` is gone.** Both the lifecycle
  handler and the GitHub-callback path used to gate workspace creation on a
  per-workspace free-plan quota. Tenant billing has no such concept, so the
  check was dropped entirely (matches the spec's "new workspaces inherit the
  tenant balance").
- **`CreditPolicy` removed.** The only callers of
  `workspace_guards.enforce(.credit_policy = .execution_required)` were the
  deleted billing handlers. Surviving callers (`zombies/lifecycle`, `activity`,
  `workspaces/ops`, `credentials`) only used `defer access.deinit(...)` — the
  field drop is safe.
- **`BillingLifecycleEvent` deleted.** No production caller survived the
  workspace-billing teardown. Enum variant + struct dropped; test count
  flipped 14 → 13.
- **`state/entitlements.zig` deleted.** After removing the workspace-billing
  sub-dir, nothing imported it. Orphan sweep caught it; documented only in
  this log, no spec rev because it's pure dead-code removal.
