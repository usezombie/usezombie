### 2. `last_used_at` async stamping deferred to a Redis queue + zombie reaper — NOT a post-flush hook

**Decision:** Amend the spec to defer the entire `last_used_at` stamping mechanism to a follow-on workstream. In the meantime, the `core.api_keys.last_used_at` column is provisioned by DDL but the middleware never writes to it; the column stays NULL until the stamping workstream ships. Intended design: bounded Redis queue enqueue on auth, drained by a zombie worker.

**Why:** The prior session's spec had pinned the mechanism to a `httpz` post-flush hook, which I flagged would require an httpz fork or a fragile middleware wrapping `res.write()` (httpz doesn't expose an `onAfterFlush` callback natively). The user chose to defer entirely — cleaner than shipping a fragile mechanism just to keep the column populated.

**How to apply in future sessions:** Any time a spec's response-path design depends on the HTTP library emitting a callback the library doesn't actually emit, surface the gap early. Don't let "post-flush hook" go un-challenged if the library doesn't have one. I noticed this only because I'd asked myself "how would I actually implement this" before executing — 30 minutes of httpz source reading would have confirmed it, but the user's deferral is faster.

**Spec amendments:** Added a deferral block to Out of Scope describing the intended Redis-queue design. Five rows across the spec annotated `(DEFERRED — see Out of Scope)`:
- §5 event catalog: `api_key.auth_succeeded` field `last_used_at_write_enqueued` dropped
- §5 event catalog: `api_key.last_used_update_failed` row annotated DEFERRED

**Decision:** 1M-key collision test runs as part of `make test` with N=100_000 in Debug and N=1_000_000 in ReleaseFast, via a comptime switch. No separate `make test-entropy` target.

**Why:** The prior session's spec had gated it behind `make test-entropy` to keep the main test suite fast. User preferred inline in `make test`: "Its an extension to unit tests." When I asked for a ballpark (5-10s ReleaseFast, 15-30s Debug at 1M) the user picked Option B: 100k in Debug, 1M in Release.
