## 4. Slice 11c remaining work — the next session's job

Three SSE-streaming tests sit between us and CHORE(close). They share an "SSE test client" need that none of the existing fixtures cover. Rough sketch:

### 4.1 SSE test client fixture

A lightweight HTTP client that opens `GET /v1/.../events/stream` with a Bearer header and yields parsed SSE frames `{ id, type, data }` to a callback. Mirrors `streamGet` in `zombiectl/src/lib/sse.js` but in Zig — needs:

- `std.http.Client` (or a raw socket if the http client misbehaves under chunked SSE — see prior incident with 413 in the spec test_event_payload_size_limit).
- Per-frame parser that emits a callback. Same shape as `parseSseBuffer` in `zombiectl/src/lib/sse-parser.js`.
- Disconnection control so reconnect tests can drop the socket mid-event.

Probably lives at `src/zombie/test_sse_client.zig` (~200 lines).

### 4.2 The 3 spec tests this unblocks

| Spec test | Approach |
| --- | --- |
| `test_sse_publish_latency_p95_lt_200ms` | Spin up the API server (or just the SSE handler) bound to a local port. Open SSE client. Worker PUBLISHes a frame; record arrival timestamp. 100 trials, compute p95, assert < 200ms. |
| `test_sse_reconnect_backfills_via_events` | Open SSE, receive frames 0..N, force-close the client socket mid-event, reconnect via `GET /events?since=<id>` for the durable backfill, then re-open SSE. Assert no event missing in the operator-visible state. |
| `test_sse_sequence_resets_on_reconnect` | Open SSE, observe frames id=0..N, disconnect, reconnect. Assert first frame on the new connection has id=0. The handler must ignore `Last-Event-ID`. |

### 4.3 Spinning up the API server in tests

This is the real complication. Today the integration tests run inside the Zig test process; they don't bring up a `zombied` HTTP server. Options:

1. **Spawn `zombied` as a child process** (mirrors the harness binary fixture). Tests connect to its TCP port. Heavy: needs binary build + signal cleanup + port allocation.
2. **Initialize the handler in-process** and use `std.http.Server` to host it on `127.0.0.1:0`. Lighter; reuses the project's existing handler dispatch. Look at `src/http/handlers/zombies/events_stream.zig` for the entry point — see if it can be wired into a stand-alone `httpz` server in a test. Probably yes; the dashboard SSE handler is small.

Recommendation: **(2)** — in-process server bound to an ephemeral port. Avoids the binary spawn overhead and CI complexity. Pattern: similar to the executor's `transport.Server` already used in `src/executor/integration_test.zig`.

## 10. Things this session learned (don't repeat)

- **Zig's test runner treats `log.err` calls during a test as test failures.** The `executor_client.connect` log.err makes any version-mismatch test fail at the framework level even when the error is the expected outcome. Workaround: bypass `ExecutorClient` and drive `transport.Client.connect` directly (mirrors `executor/crash_test.zig`).
- **Bun's test filter (`-Dtest-filter` for Zig, file-level for Bun) does substring matching.** Zig's filter applies to the fully-qualified test name (file path + `test.<name>`). When a filter doesn't match anything, the suite still runs the 18 baseline `test {}` import blocks — that's normal, not a flag of skipped work.
- **`std.json.Parsed.deinit` reclaims an arena.** Returning `parsed.value` only (and dropping the `Parsed` wrapper) leaks the entire parse buffer. If a function takes ownership of a parsed JSON via parseFromSlice, it must return the full `std.json.Parsed(T)` (or own + deinit the Parsed via its own lifetime).
- **`std.process.Child.kill` (Zig 0.15.2 on POSIX) sends SIGKILL and reaps the child internally** via waitUnwrappedPosix. No need to call `wait()` after kill; doing so would no-op or error.
- **The harness binary's `GeneralPurposeAllocator` reports leaks on shutdown.** These leaks are pre-existing in `executor/main.zig` (env var reads via `getEnvVarOwned` → `dupe`, never freed). They're harmless to test outcomes but fill stderr. Do not "fix" by adding test-only paths to `main.zig`.
- **Redis pub/sub-only failure injection has no clean primitive.** The cleanest production-code-safe path is a Redis ACL user without `+publish` permission. Adding a test-only flag to `queue_redis.Client` is production pollution and was rejected.
- **`activity_publisher.KIND_*` are `pub const` now.** Tests assert against the symbol, never the literal. RULE UFS pre-edit grep should catch any re-stringification.
- **Bun zombiectl tests run via `make test-unit-zombiectl`** in the `test-zombiectl` GitHub Actions job. CLI integration tests (mocked deps) belong here, not in `make test-integration` which is Zig-only.
- **The harness fixture requires `wait()` after `kill()` for Child handle cleanup** — without it, the Zig test allocator detected leaks at suite shutdown. Already fixed in `terminateChild`.
