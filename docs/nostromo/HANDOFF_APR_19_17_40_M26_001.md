## 1 · The standard (locked-in, apply everywhere)

After Apr 19's back-and-forth, this is the single standard across the codebase:

| Pattern | Approach |
|---|---|
| Reusable primitive with variants/logic (Button, Badge, Card, Dialog, Tooltip) | **DS component** — lives in `ui/packages/design-system/src/design-system/` |
| Layout pattern repeated in 2+ places (PageHeader, PageTitle, SectionLabel) | **DS component** — same location, imported from the barrel |
| One-off layout on a single component | **Inline Tailwind utilities** on the JSX — acceptable because non-reused |
| Domain-specific viz with no external consumer (PipelineStage, RunStatus, ArtifactRow, TransitionRow) | **App-local domain component** in `ui/packages/app/components/domain/`, consumes DS primitives |
| Design tokens (colors, spacing, shadows) | **Semantic Tailwind utilities via `@theme inline`** — `bg-primary`, `text-muted-foreground`, `border-border`. Never raw `var(--z-*)` in consumer JSX |
| Text sizing | **Preset scale only** — `text-xs` / `text-sm` / `text-base` / `text-lg` / `text-xl` / `text-2xl`. Never arbitrary `text-[0.7rem]` |
| CSS `@apply` | **Do not introduce.** `@apply`-backed CSS classes are a worse form of utility-duplication — lose type-safety, split the vocabulary between CSS and JSX |
| Repeated utility stacks in consumer JSX | **Promote to a DS component.** If you're copy-pasting the same 4+ utility classes across files, that's a component |

**Invariant greps that still must stay empty** (from `HANDOFF_APR_19_10_58_M26_001.md` §1.4):
```bash
grep -rn "^\.z-\|^ *\.z-" ui/packages --include="*.css" | grep -v node_modules
grep -rn '"z-[a-z]' ui/packages --include="*.tsx" --include="*.ts" | grep -v node_modules | grep -v "\-\-z-\|var(--z-" | grep -v ".test."
grep -rn "forwardRef" ui/packages/design-system/src | grep -v "no forwardRef"
grep -rn "var(--z-" ui/packages/design-system/src/design-system/*.tsx | grep -v "ZombieHandIcon\|.test."
grep -rn "googleapis\|jsdelivr\|cdn\." ui/packages --include="*.css" --include="*.tsx" --include="*.ts" | grep -v node_modules | grep -v "/dist/"
grep -rn 'from "react-router-dom"\|from "next/link"\|from "next/navigation"' ui/packages/design-system/src
```

Add these two to the invariants set for this milestone's doctrine going forward:
```bash

# No @apply in app/website stylesheets (DS package tokens.css + theme.css are exempt)
grep -rn "@apply " ui/packages/app ui/packages/website --include="*.css" | grep -v node_modules
```

Both must stay empty on this branch.

## 6 · If something is off

- **Arbitrary text size leaks back in** — likely a copy-paste from an older page. Replace with the closest preset size.
- **Inline utility stack duplicated across 2+ files** — promote to a DS component. Do not `@apply`.
- **`var(--z-*)` in consumer JSX** — the Layer 2 semantic token for it probably exists (see `ui/packages/design-system/src/theme.css`'s `@theme inline` block). Use that utility instead.
- **App build fails with "useState in Server Component"** — a DS component picked up a hook and lost its `"use client"` directive, or a consumer imported a client-only DS component into a page without the directive. Check the stack trace for the file; add the directive.
- **Playwright Mission Control hover flaky again** — the button variant was switched to `ghost` in commit `260185f3`. If someone flips it back to `default`, `.header-mission-control:hover`'s border-color override will be out-precedenced by `border-transparent`. Keep `variant="ghost"`.
