## Pre-push hook block (and fix)

When I ran `git push -u origin feat/m37-platform-ops`, the pre-push hook ran the full test matrix. `zombiectl` passed (270 tests, 366 expect calls). Website package tripped on `vitest: command not found` with `error: script "test" exited with code 127`.

Memory entry `reference_bun_workspace_install.md` says: root `bun install` hydrates `ui/packages/{website,app}` vitest; a missing vitest means the workspace was never installed. I ran `bun install` at the worktree root (exit 0), then re-pushed cleanly.

**Lesson (and a possible future rule):** a fresh worktree on any repo with a bun workspace needs `bun install` at root before the first push that triggers the full test hook. Worth considering a CI or pre-commit warning that prompts this, since the failure mode — tests silently skipped as "not installed" — is easy to misread as "flaky hook, re-run."
