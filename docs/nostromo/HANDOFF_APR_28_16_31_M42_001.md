## 4. Decisions made — don't re-litigate

1. **Slice 7 (admin-resume fallback) is dropped from M42.** Originally a temporary endpoint so M42 could ship before M47 was ready; per scope decision Apr 28, 2026 it's not worth building since M47's Approval Inbox owns resolution. Stranded `gate_blocked` rows remain visible via `GET /events`; only the resume action waits on M47. Spec already updated.
2. **SSE auth is Bearer for CLI/API, cookie for browser dashboard.** Cookie middleware does NOT yet exist in the repo and lands with slice 10 phase 2. CLI/dashboard split is industry-standard (GitHub, Stripe, Vercel, Linear, Anthropic). Cookies-everywhere was rejected (CSRF, no Node cookie jar, server-to-server unfit).
3. **Clerk is the OIDC provider.** `clerk.dev.usezombie.com` is the dev environment hostname — not static, env-dependent. Server reads JWKS URL from `OIDC_JWKS_URL` config; `jwks.zig` caches with 6 h TTL and refreshes on kid miss (rotation-safe).
4. **`session_id` in `/v1/auth/sessions/{id}`** is a CLI device-flow handshake row (5 min TTL, deleted after complete). Not a user session — that lives in the Clerk JWT.
5. **No emailless signup path.** GitHub OAuth (the only Clerk connector wired) requires email; if you ever need it, add a different Clerk connector first.
6. **Dashboard RecentActivity failure mode flipped.** Used to return `null` and hide the panel on fetch failure; now renders `EventsList` with empty initial page so operators see section + EmptyState.
7. **Slice 10 phase 1 was reworked once.** First attempt was a compatibility shim that mapped `EventRow → ActivityEvent`. User correctly rejected it as not the right pattern. The shipped version exposes the native `EventRow` shape and uses design-system primitives throughout.
8. **`zombiectl steer` interactive REPL deferred.** Batch mode handles 99% of operator and chat-UI integration; REPL is mostly UX nicety. File under follow-up if prioritized.

## 9. Things this session learned (don't repeat)

- The dashboard's `/activity` endpoints have been 404ing since slice 8 — fixed in slice 10 phase 1. If you see any other `/activity` reference in the codebase outside `ActivityIcon`, it's stale; convert to `/events`.
- `URLSearchParams` keeps `*` literal (sub-delim, not encoded). My first `events.test.ts` asserted `actor=webhook%3A%2A` and failed — the actual encoding is `actor=webhook%3A*`.
- `@usezombie/design-system` does not export a List/Timeline primitive. `EventsList` composes `Card` + `Badge` + `Pagination` for now. If you find yourself wanting a generic `<DataList>` again (telemetry, grants, etc.), spec out a design-system primitive workstream rather than re-implementing the pattern.
- `bun run vitest --run` in `ui/packages/app/` globs into `node_modules/` and reports failures from `next/dist/...`, `zod/src/...`, etc. Pre-existing config drift, unrelated to your work. Run targeted file lists when verifying changes (e.g. `vitest --run lib/api/events.test.ts dashboard-coverage coverage-edges`).
- Pre-push hook is opinionated. It runs `make lint` + multiple test targets before pushing. Don't rely on `zig build test` alone — the executor test target catches signature mismatches the worker build doesn't.
- `var msg_opt = try …` triggers "local variable is never mutated" in Zig 0.15.2. Use `const` when not reassigning.
- The `Section` design-system primitive renders a `<div>` by default; use `asChild` to keep semantic `<section>`. The codebase had raw `<section className="mb-8">` everywhere before slice 10 — converted all four dashboard pages.
