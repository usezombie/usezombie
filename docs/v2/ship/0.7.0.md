# v0.7.0

**Date:** Apr 10, 2026
**Milestone:** M5_001
**Spec:** M5_001_TOOLS_AS_ATTACHMENTS.md

## What changed

- Executor tool invocation is now table-driven: adding an executor-side tool requires one registry entry in `tool_bridge.zig`, zero changes elsewhere.
- New vtable interface for skill-backed tools (`src/zombie/tools/tool.zig`): all future API tools (Slack, GitHub, etc.) implement this contract — credential is pre-injected by the bridge, never passed by the tool.
- Credential echo stripping built in: tool responses are scanned for raw and base64-encoded credential values before reaching the agent.
- Three unreachable orphaned test files deleted (`proposals_test.zig` breadcrumb, `gate_loop_integration_test.zig`, `topology_gate_test.zig`).

## Technical details

- `src/executor/tool_bridge.zig`: replaces hardcoded if/else in `runner.buildToolsFromSpec()`. Static `BRIDGE_REGISTRY` table + `resolve(name)` + `buildTools(alloc, spec, workspace, cfg)` + `stripCredentialEcho(alloc, output, cred)`.
- `src/zombie/tools/tool.zig`: vtable `Tool` type + `ToolResult` + `ToolError` + `ToolVTable(T)` comptime helper. No concrete implementations (those ship per-skill in future milestones).
- `src/executor/runner_credentials.zig`: extracted from `runner.zig` to satisfy 400-line gate. Holds `prepareGitCredential()` (was `configureGitCredentials`). Future tool credential setups go here.
- `runner.zig` reduced from 481 → 393 lines.
- 12 new executor unit tests (tool_bridge tests). Total: 350 (was 338).

## Breaking changes

- None. The executor RPC protocol is unchanged; the tool spec array format is unchanged.

## Migration

- None. Server and executor update together as a single deployment unit.
