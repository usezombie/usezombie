# v0.7.0

**Date:** Apr 10, 2026

## What changed

- Zombies are now a directory (`SKILL.md` + `TRIGGER.md`) instead of a single file — `zombiectl install` and `zombiectl up` handle this automatically.
- `SKILL.md` is now publishable to the ClaHub registry as-is.
- Webhook URLs can embed the secret directly: `POST /v1/webhooks/{id}/{secret}` — no Authorization header required.
- Credentials in `TRIGGER.md` are referenced by name (e.g. `agentmail_api_key`), not raw `op://` paths.
- Zombie chains: use `chain:` in `TRIGGER.md` to forward events to downstream zombies.
- Skills use built-in tools (shell, HTTP, file read) from `SKILL.md` instructions — no custom Zig needed per skill.

## Breaking changes

- `POST /v1/zombies/` now expects `trigger_markdown` (the contents of `TRIGGER.md`) instead of `config_json`.
- `POST /v1/webhooks/{zombie_id}/{secret}` is the preferred webhook format — Bearer fallback still works.
- Credential values in `TRIGGER.md` must be names, not `op://` paths.

## Migration

- Tables rebuilt from scratch (pre-v1.0 policy). No migration needed.
