# v0.8.0

**Date:** Apr 11, 2026
**Milestone:** M6_001
**Spec:** M6_001_AI_FIREWALL_POLICY_ENGINE.md

## What changed

- Every outbound request from a Zombie now passes through a 4-layer AI Firewall before reaching external APIs
- **Domain allowlist** ‚Äî only domains declared in the Zombie's skills can be reached (defense-in-depth on top of network policy)
- **Endpoint policy** ‚Äî per-endpoint rules in TRIGGER.md `firewall:` section (e.g., "Stripe: allow GET /v1/charges, deny POST /v1/refunds without approval")
- **Prompt injection detection** ‚Äî scans outbound request bodies for instruction override, role hijacking, data exfiltration, and jailbreak patterns (with unicode normalization)
- **Content scanning** ‚Äî inspects response bodies for credential leakage, PII (credit cards, SSNs), and API key patterns from other services
- Every firewall decision (allow, block, flag) is logged as a structured event in the activity stream
- Fail-closed on all error paths ‚Äî if the firewall errors, the request is denied

## Technical details

- New module: `src/zombie/firewall/` with 6 files (946 lines total):
  - `domain_policy.zig` (66L) ‚Äî case-insensitive exact domain match, no suffix matching
  - `endpoint_policy.zig` (221L) ‚Äî glob-based path matching, JSON rule parser
  - `injection_detector.zig` (175L) ‚Äî 19 patterns across 4 categories, unicode `\uXXXX` normalization
  - `content_scanner.zig` (244L) ‚Äî credit card, SSN, API key prefix detection, credential echo check, 1MB scan limit
  - `firewall.zig` (138L) ‚Äî orchestrator combining all 4 layers with event type mapping
  - `firewall_test.zig` (110L) ‚Äî orchestrator integration tests
- Uses `union(enum)` for `FirewallDecision`, `InjectionResult`, `ScanResult` ‚Äî exhaustive switch enforcement
- Firewall events reuse existing `core.activity_events` table with new event_type values: `firewall_request_allowed`, `firewall_request_blocked`, `firewall_injection_detected`, `firewall_content_flagged`, `firewall_approval_triggered`
- Pattern matching is O(n*m) substring scan ‚Äî target < 1ms for 10KB bodies

## Breaking changes

- None. Firewall is additive ‚Äî existing tool bridge and activity stream interfaces unchanged.

## Migration

- None ‚Äî no new tables. Firewall events use existing `core.activity_events` with new event_type values.
