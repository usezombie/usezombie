# Test Plan
Generated by /plan-eng-review on Mar 22, 2026
Branch: main

## Affected Pages/Routes
- `zombiectl` routed commands (`login`, `workspace add`, `run`, unknown command handling) — verify analytics remains non-blocking and uses the correct distinct ID semantics.
- CLI state persistence (`~/.zombie/*` today, future XDG migration target) — verify auth/session state remains separate from analytics configuration.

## Key Interactions to Verify
- Successful `zombiectl login` emits command lifecycle events and `user_authenticated` with a post-login distinct ID.
- Successful `zombiectl workspace add` emits `workspace_created` without requiring analytics state in persisted auth files.
- Successful `zombiectl run` emits `run_triggered` without breaking command execution when analytics is unavailable.
- Unknown commands and routed failures emit `cli_error` without changing CLI exit behavior.

## Edge Cases
- Analytics disabled: no client created, no external calls, command behavior unchanged.
- Analytics dependency missing/import failure: CLI continues with no crash.
- Missing or malformed stored auth token: analytics falls back to `anonymous`.
- Proposed future state-root migration to `~/.config/zombiectl/*`: auth persistence migration must not accidentally pull analytics key into auth storage.

## Critical Paths
- `zombiectl login` -> credentials persisted -> post-login distinct ID extracted -> analytics shutdown completes without blocking exit.
- `zombiectl workspace add` -> API success -> workspace persistence -> telemetry event capture remains fire-and-forget.
- `zombiectl run` -> API success -> telemetry event capture remains fire-and-forget even on network/SDK failure.
