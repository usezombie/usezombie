## Affected Pages/Routes
- `/v1/runs` — verify free-plan admission includes `plan_tier`, `billing_status`, and remaining credit fields in the response.
- `/v1/workspaces/:id:sync` — verify free-plan credit exhaustion blocks sync and successful sync returns remaining credit metadata.
- `/v1/workspaces/:id/harness/source` — verify operator role is required and free-plan exhaustion blocks execution-affecting harness writes.
- `/v1/workspaces/:id/harness/compile` — verify operator role is required and exhausted free-plan workspaces receive deterministic credit rejection.
- `/v1/workspaces/:id/harness/activate` — verify operator role is required.
- `/v1/workspaces/:id/skill-secrets/*` — verify operator role is required.
- `/v1/agents/:id` and `/v1/agents/:id/scores` — verify operator role is required.
- `/v1/workspaces/:id/scoring/config` — verify operator role is required and token bounds validation is deterministic.
- `/v1/workspaces/:id/billing/scale` — verify operator role is required and successful upgrade returns active Scale billing state.
- `/v1/workspaces/:id/billing/events` — verify admin role is required for billing lifecycle mutations.
- `/pricing` — verify free-tier `$10` no-expiry copy and direct Scale upgrade CTA remain aligned with backend behavior.

## Key Interactions to Verify
- Start a run on a free workspace and confirm CLI prints `plan_tier`, `credit_remaining_cents`, and `credit_currency`.
- Sync specs on a free workspace with credit remaining and confirm CLI prints credit balance.
- Exhaust free credit and confirm CLI prints the `workspace upgrade-scale` command.
- Open `zombiectl --help` with a user token and verify operator commands are hidden.
- Open `zombiectl --help` with an operator token and verify harness, skill-secret, agent, and admin-config surfaces are shown as intended.
- Use `zombiectl workspace upgrade-scale --workspace-id ... --subscription-id ...` and confirm the billing endpoint is called and human output reflects active Scale state.

## Edge Cases
- Role claim absent: server defaults JWT users to `user`.
- Role claim malformed or unknown: server rejects the token for protected operator/admin surfaces.
- API key auth: treated as `admin`.
- Free credit reaches zero exactly after a completed billable run: next execution attempt is rejected before admission.
- Failed or incomplete runs after stage usage recording: no free-credit deduction occurs.
- Unix-socket transport startup race: test retries connection instead of relying on a fixed sleep.

## Critical Paths
- Free workspace creation -> run admitted -> completed runtime billed -> credit balance decreases -> next exhausted run rejected.
- Operator token -> harness/skill-secret/agent/scoring operations allowed -> user token denied with deterministic insufficient-role behavior.
- Exhausted free workspace -> CLI shows Scale upgrade path -> operator upgrades workspace -> subsequent execution proceeds under Scale billing state.
